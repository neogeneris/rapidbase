RapidBase Framework

Manual de Usuario - Guía Completa de Desarrollo

📖 Visión General

RapidBase es un framework PHP ligero y moderno diseñado para acelerar el desarrollo de aplicaciones web con base de datos. Proporciona una capa de abstracción potente sobre PDO con características avanzadas como:

  • ORM ActiveRecord - Patrón familiar para trabajar con entidades
  • Consultas Fluent - API intuitiva para operaciones CRUD
  • JOINs Automáticos - Relaciones configurables entre tablas
  • Caché Multi-nivel - Optimización automática de consultas
  • Streaming - Manejo eficiente de grandes volúmenes de datos
  • Eventos - Sistema de hooks para extensibilidad
💡 Tip: RapidBase está diseñado para ser utilizado tanto de forma procedural (clase DB) como orientada a objetos (clase Model).

🏗️ Arquitectura

RapidBase sigue una arquitectura en capas bien definida:

DB (Fachada)
↓
Gateway
↓
SQL Builder
↓
Executor
↓
Conn (PDO)

Componentes Principales

Clase Responsabilidad
DB Fachada principal. Punto de entrada único para todas las operaciones.
Gateway Orquestador. Combina SQL, Conn, Executor y Caché.
SQL Constructor de consultas. Genera SQL seguro con soporte para JOINs.
Executor Ejecutor atómico. Ejecuta sentencias y maneja errores.
Conn Pool de conexiones. Administra instancias PDO.
Model ORM ActiveRecord. Representa entidades de negocio.
CacheService Gestión de caché. Soporta múltiples adapters.

📦 Instalación

Requisitos mínimos:

  • PHP 8.0 o superior
  • Extensión PDO habilitada
  • Base de datos MySQL, PostgreSQL, SQLite o compatible

Vía Composer

composer require rapidbase/framework

Estructura del Proyecto

│   └── rapidbase/
│       └── src/

🔌 Conexión a Base de Datos

La configuración inicial se realiza mediante el método DB::setup():

Ejemplo MySQL

<?php
use RapidBase\Core\DB;

// Configuración de conexión
DB::setup(
    dsn: 'mysql:host=localhost;dbname=mi_base;charset=utf8mb4',
    user: 'usuario',
    pass: 'contraseña',
    name: 'main'  // Nombre de la conexión (para múltiples conexiones)
);

Ejemplo SQLite

DB::setup(
    dsn: 'sqlite:/path/to/database.sqlite',
    user: '',
    pass: ''
);

Múltiples Conexiones

// Conexión principal
DB::setup('mysql:host=localhost;dbname=app', 'user', 'pass', 'main');

// Conexión secundaria
DB::setup('mysql:host=localhost;dbname=analytics', 'user', 'pass', 'analytics');

// Seleccionar conexión
DB::getConnection('analytics');
⚠️ Importante: La configuración debe realizarse antes de cualquier operación de base de datos.

💾 Configurar Caché

RapidBase incluye un sistema de caché multi-nivel que puede mejorar significativamente el rendimiento:

use RapidBase\Core\Cache\CacheService;

// Inicializar caché en directorio local
CacheService::init('/path/to/cache/dir');

// El caché se activa automáticamente para consultas frecuentes
$result = DB::all('users');  // Primera vez: DB, siguientes: caché

Adapters Disponibles

  • DirectoryCacheAdapter - Archivos locales (default)
  • RedisCacheAdapter - Redis server
  • MemcachedCacheAdapter - Memcached
  • SQLiteMemoryCacheAdapter - SQLite en memoria

🔗 Mapa de Relaciones

Para habilitar JOINs automáticos, define un mapa de relaciones:

// config/relationships.php
return [
    'relationships' => [
        'from' => [
            'orders' => [
                'users' => ['local_key' => 'user_id', 'foreign_key' => 'id']
            ],
            'order_items' => [
                'products' => ['local_key' => 'product_id', 'foreign_key' => 'id']
            ]
        ],
        'to' => [
            'users' => [
                'orders' => ['local_key' => 'id', 'foreign_key' => 'user_id']
            ]
        ]
    ],
    'tables' => [
        'users' => ['id' => 'int', 'name' => 'string', 'email' => 'string'],
        'orders' => ['id' => 'int', 'user_id' => 'int', 'total' => 'decimal']
    ]
];

Cargar el Mapa

DB::loadRelationsMap('config/relationships.php');

// O programáticamente
DB::setRelationsMap($mapArray);
💡 Tip: Usa SchemaMapper::generate() para generar automáticamente el mapa desde tu base de datos.

🎯 Clase DB - Inicialización

La clase DB es la fachada principal del framework. Todos los métodos son estáticos:

use RapidBase\Core\DB;

// Setup inicial
DB::setup($dsn, $user, $pass);

// Cargar relaciones
DB::loadRelationsMap('schema.php');

📖 DB - Operaciones de Lectura

Obtener un registro

// Buscar por condiciones
$user = DB::find('users', ['email' => 'juan@example.com']);

// Contar registros
$count = DB::count('users', ['active' => 1]);

// Verificar existencia
$exists = DB::exists('users', ['email' => 'test@test.com']);

Obtener múltiples registros

// Todos los registros (con caché)
$users = DB::all('users');

// Con condiciones y ordenamiento
$activeUsers = DB::all(
    table: 'users',
    conditions: ['active' => 1],
    sort: ['name' => 'ASC']
);

// Paginado (retorna array llave=>valor si hay 2+ columnas)
$userList = DB::list(
    table: 'users',
    where: ['active' => 1],
    sort: ['created_at' => 'DESC'],
    page: 1,
    perPage: 20
);

Consultas SQL personalizadas

// Una fila
$row = DB::one('SELECT * FROM users WHERE id = ?', [1]);

// Múltiples filas
$rows = DB::many('SELECT * FROM users WHERE active = ?', [1]);

// Valor escalar
$count = DB::value('SELECT COUNT(*) FROM users');

// Fetch simple
$data = DB::fetch('SELECT * FROM orders WHERE total > ?', [100]);

Grid para DHTMLX

use RapidBase\Core\QueryResponse;

$response = DB::grid(
    table: 'products',
    where: ['category_id' => 5],
    sort: ['name' => 'ASC'],
    page: 1,
    perPage: 50
);

// Retorna QueryResponse con data, total, count, metadata
echo json_encode($response->toDhtmlx());

✏️ DB - Operaciones de Escritura

Insertar registros

// Insertar y obtener ID
$id = DB::insert('users', [
    'name' => 'Juan Pérez',
    'email' => 'juan@example.com',
    'password' => password_hash('secret', PASSWORD_DEFAULT)
]);

// Insertar o false si falla
$result = DB::create('users', [
    'name' => 'María García',
    'email' => 'maria@example.com'
]);

Actualizar registros

$success = DB::update(
    table: 'users',
    data: ['name' => 'Juan Carlos Pérez'],
    conditions: ['id' => 1]
);

Eliminar registros

$deleted = DB::delete('users', ['id' => 1]);

Upsert (Insertar o Actualizar)

// Si existe (por email), actualiza. Si no, inserta.
$result = DB::upsert(
    table: 'users',
    data: ['name' => 'Nuevo Nombre'],
    identifier: ['email' => 'juan@example.com']
);

// Retorna: lastId si insertó, true si actualizó, false si falló

Lectura como Objeto

// Leer y mapear a instancia de clase
$user = DB::readAs(
    class: User::class,
    where: ['id' => 1],
    table: 'users'  // opcional si User tiene getTable()
);

🚀 DB - Métodos Avanzados

Estado de la última operación

$status = DB::status();
// ['sql' => '...', 'params' => [...], 'rows' => 5, 'id' => 123, 'error' => null]

SQL Crudo

// Ejecutar sentencia directa
DB::exec('DELETE FROM logs WHERE created_at < ?', [date('Y-m-d', strtotime('-30 days'))]);

// Query directa
$stmt = DB::query('SELECT * FROM large_table');
while ($row = $stmt->fetch(PDO::FETCH_ASSOC)) {

}

// Expresión raw (evita escape)

    'title' => 'Mi Post',
    'created' => DB::raw('NOW()')
]);

Streaming para Grandes Volúmenes

// Iterar sin cargar todo en memoria
foreach (DB::stream('SELECT * FROM telemetry_data') as $row) {
    process($row);
}

🏛️ ORM - Definición de Modelos

Los modelos extienden RapidBase\ORM\ActiveRecord\Model:

<?php
namespace App\Models;

use RapidBase\ORM\ActiveRecord\Model;

class User extends Model
{
    protected static string $table = 'users';
    protected static string $primaryKey = 'id';
    
    // Propiedades adicionales (opcional)
    public string $name = '';
    public string $email = '';
}

Modelo con Métodos Personalizados

class Product extends Model
{
    protected static string $table = 'products';
    
    public function getFormattedPrice(): string
    {
        return '$' . number_format($this->price, 2);
    }
    
    public function isActive(): bool
    {
        return $this->status === 'active';
    }
}

🏛️ ORM - Operaciones CRUD

Crear

// Crear instancia y guardar
$user = new User(['name' => 'Juan', 'email' => 'juan@test.com']);
$user->save();

// Creación estática
$id = User::create([
    'name' => 'María',
    'email' => 'maria@test.com'
]);

Leer

// Por ID
$user = User::read(1);

// Todos
$users = User::all();

// Acceder a propiedades
echo $user->name;
echo $user['email'];  // También funciona

Actualizar

$user = User::read(1);
$user->name = 'Juan Carlos';
$user->email = 'jc@test.com';
$user->save();  // Solo actualiza campos modificados

Eliminar

// Por ID
User::delete(1);

// Desde instancia
$user = User::read(1);
$user->destroy();

🏛️ ORM - Gestión de Estado

$user = User::read(1);

// Verificar si está "sucio" (modificado)
if ($user->isDirty()) {
    echo "El registro ha sido modificado";
}

// Verificar campo específico
if ($user->isDirty('email')) {
    echo "El email cambió";
}

// Obtener solo campos modificados
$changes = $user->getDirty();
// ['email' => 'nuevo@email.com']

// Sincronizar estado original
$user->syncOriginal();
ℹ️ Nota: El ORM detecta automáticamente qué campos han cambiado y solo actualiza esos campos en la base de datos.

🔧 Gateway - Consultas SELECT

Gateway es la capa intermedia que puedes usar directamente para más control:

use RapidBase\Core\Gateway;

// Consulta básica
$result = Gateway::select(
    fields: '*',
    table: 'users',
    where: ['active' => 1],
    sort: ['name' => 'ASC'],
    page: 1,
    perPage: 20
);

// Estructura de retorno
/*
[
    'data' => [...],      // Array de resultados
    'total' => 150,       // Total de registros
    'page' => 1,          // Página actual
    'limit' => 20,        // Registros por página
    'source' => 'database',
    'timestamp' => ...
]
*/

Consulta con GROUP BY

$result = Gateway::select(
    fields: ['category', 'COUNT(*) as total', 'AVG(price) as avg_price'],
    table: 'products',
    where: ['active' => 1],
    groupBy: ['category'],
    having: ['total' => ['>' => 5]],
    sort: ['total' => 'DESC']
);

🔧 Gateway - JOINs Automáticos

Cuando pasas un array de tablas, Gateway construye los JOINs automáticamente usando el mapa de relaciones:

// JOIN automático entre users y orders
$result = Gateway::select(
    fields: ['users.name', 'orders.total', 'orders.created_at'],
    table: ['users', 'orders'],  // Array de tablas
    where: ['users.active' => 1],
    sort: ['orders.created_at' => 'DESC']
);

// SQL generado automáticamente:
// SELECT `users`.`name`, `orders`.`total`, `orders`.`created_at`
// FROM `users`
// INNER JOIN `orders` ON `users`.`id` = `orders`.`user_id`
// WHERE `users`.`active` = ?
// ORDER BY `orders`.`created_at` DESC
// LIMIT 10 OFFSET 0
💡 Tip: El algoritmo optimiza el orden de los JOINs basándose en las relaciones definidas.

🔧 Gateway - Caché de Consultas

// selectCached usa caché automáticamente
$result = Gateway::selectCached(
    fields: '*',
    table: 'products',
    where: ['category' => 'electronics'],
    ttl: 3600  // 1 hora
);

// La primera vez consulta la BD
// Las siguientes veces (dentro del TTL) retorna de caché
// source será 'cache' en lugar de 'database'

Invalidación Automática

Cuando ejecutas INSERT, UPDATE o DELETE, Gateway invalida automáticamente el caché de esa tabla.

Control Manual del Caché SQL

// Habilitar/deshabilitar caché de consultas SQL
Gateway::setSqlQueryCacheEnabled(true);

// Ver estadísticas
$stats = Gateway::getCacheStats();
/*
[
    'sql_query_cache' => ['hits' => 100, 'misses' => 20, 'hit_rate' => 0.83],
    'result_cache' => [...]
]
*/

💫 Transacciones

use RapidBase\Core\DB;

DB::transaction(function() {
    // Todas estas operaciones son atómicas
    DB::insert('orders', ['user_id' => 1, 'total' => 100]);
    DB::insert('order_items', ['order_id' => 1, 'product_id' => 5, 'qty' => 2]);
    DB::update('products', ['stock' => DB::raw('stock - 2')], ['id' => 5]);
    
    // Si alguna falla, todo se revierte
});
⚠️ Importante: Si se lanza una excepción dentro del callback, la transacción se revierte automáticamente.

🌊 Streaming

Para procesar grandes volúmenes de datos sin agotar la memoria:

use RapidBase\Core\DB;

// Generador que itera fila por fila
foreach (DB::stream('SELECT * FROM massive_telemetry_table') as $row) {
    processRow($row);
    // Solo una fila en memoria a la vez
}

// También disponible en Executor
use RapidBase\Core\Executor;
foreach (Executor::stream($sql, $params) as $row) {
    // procesar
}

📢 Sistema de Eventos

RapidBase dispara eventos que puedes escuchar:

use RapidBase\Core\Event;

// Escuchar evento específico
Event::listen('db.success', function($data) {
    error_log("Query exitosa: " . $data['sql']);
});

// Escuchar errores
Event::listen('db.error', function($data) {
    error_log("Error DB: " . $data['error']);
});

// Escuchar todos los eventos
Event::listen('*', function($eventName, $data) {
    logEvent($eventName, $data);
});

// Disparar evento manualmente
Event::fire('custom.event', ['message' => 'Hola']);

Eventos Disponibles

  • db.success - Consulta exitosa
  • db.error - Error en consulta
  • db.log - Log de toda operación
  • * - Comodín para todos los eventos
